## 技術スタック

| レイヤー | 採用技術 | 理由 |
|---|---|---|
| フロントエンド | Next.js（React） | AIとの相性が良い・将来移行しやすい |
| バックエンド | Next.js API Routes | フロントと一体管理でシンプル |
| データベース | Supabase（PostgreSQL） | 将来の自社PostgreSQL移行が容易 |
| 認証 | Supabase Auth | メール+PW・パスワードリセットがそのまま使える |
| メール送信 | Resend | 月3,000通まで無料・導入が簡単 |
| ホスティング | Vercel | Next.jsとの相性が最良・自動デプロイ対応 |

## Phase1 インフラ構成

```
スタッフのスマホ・iPad・管理者のPC
        ↓（ブラウザでアクセス）
    Vercel（Next.js）
    ├── 画面（フロントエンド・React）
    └── API Routes（バックエンド）
        ↓（データのやり取り）
    Supabase（PostgreSQL）
    └── Supabase Auth（認証）
        ↓（メール送信）
    Resend
```

---

## 将来の自社サーバー移行イメージ

```
【Phase1・現在】               【将来・M1 Mac mini移行後】
Vercel          →    M1 Mac mini（Next.js をセルフホスト）
Supabase        →    M1 Mac mini（PostgreSQL をローカル運用）
Resend          →    そのまま継続 or 自前メールサーバー
```

**移行の手順イメージ:**
1. SupabaseのデータをPostgreSQLとしてエクスポート
2. M1 Mac miniにPostgreSQLをインストールしてインポート
3. Next.jsをM1 Mac miniにデプロイ
4. DNSを切り替え

大規模な作り直しは不要で、設定変更と移行作業のみで完結します。
